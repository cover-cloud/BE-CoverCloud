## 📌 프로젝트 소개

음악 커버곡을 공유하고 검색할 수 있는 플랫폼의 백엔드 시스템입니다. 
마이크로서비스 아키텍처(MSA)로 설계하여 5개의 독립적인 서비스로 구성되어 있으며, 
각 서비스가 독립적으로 확장 가능합니다. 
JWT 기반 인증, Redis 캐싱, JPA 쿼리 최적화를 통해 안전하고 성능 최적화된 시스템을 구축했습니다.

```kotlin
fun updateProfile(

}
```

```kotlin
@Transactional
fun deleteAccount(accessToken: String): String {
    val userInfo = authService.getUserInfo(accessToken)
    val userId = userInfo.userId
    val user = userRepository.findById(userId)
        .orElseThrow { IllegalArgumentException("User not found") }
    
    // Soft delete: 사용자 상태를 DELETED로 변경
    user.status = UserStatus.DELETED
    userRepository.save(user)
    
    // Refresh token 제거
    refreshTokenRepository.deleteByUserId(userId)
    
    return "Account deleted successfully"
}
```

**고려사항**:
- Soft delete로 데이터 보존 (사용자가 작성한 콘텐츠 추적 가능)
- 토큰 무효화로 즉시 로그아웃

### 5️⃣ 음악 정보 관리 (Music Service)

```kotlin
@Service
class MusicService(
    private val musicRepository: MusicRepository
) {
    // 중복 확인 후 저장
    @Transactional
    fun saveMusic(request: CreateMusicDto): MusicResponse {
        val existing = musicRepository.findByTitleAndArtist(
            request.title, 
            request.artist
        )
        
        return if (existing != null) {
            MusicResponse(
                id = existing.id,
                title = existing.title,
                artist = existing.artist
            )
        } else {
            val music = Music(title = request.title, artist = request.artist)
            val saved = musicRepository.save(music)
            MusicResponse(
                id = saved.id,
                title = saved.title,
                artist = saved.artist
            )
        }
    }
}
```

**특징**:

## 📞 면접 대비 질문과 답변

### Q. 왜 JWT 토큰을 선택했나요?
**A.** 서버 부하를 줄이고, 분산 환경에서 확장성이 좋기 때문입니다. 
Stateless 인증으로 세션 서버가 필요 없고, 
마이크로서비스 환경에서 각 서비스가 독립적으로 토큰을 검증할 수 있습니다.

### Q. 캐싱은 어떻게 구현했나요?
**A.** Spring의 @Cacheable 어노테이션으로 선언적 캐싱을 구현하고, 
Redis를 백엔드로 사용했습니다. 
자주 조회되는 데이터(커버곡, 트렌딩)에 TTL을 설정해서 
자동으로 만료되도록 했습니다.

### Q. N+1 문제를 어떻게 해결했나요?
**A.** Fetch Join을 사용하거나 @EntityGraph로 즉시 로딩을 설정했습니다.
또한 배치 사이즈를 적절히 설정해서 쿼리 횟수를 줄였습니다.

### Q. 서비스 간 통신이 실패하면?
**A.** Feign Client에 Fallback을 구현해서 기본값을 반환하거나,
로깅 후 사용자에게 친화적인 에러 메시지를 반환합니다.

### Q. 사용자 증가 시 대응 방안은?
